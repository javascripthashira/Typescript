error   → the app is broken. Something failed.
warn    → nothing broke yet, but something looks wrong
info    → normal stuff happened. User logged in. Server started.
http    → an HTTP request came in and got a response
debug   → deep detail. Only useful when you're investigating something

"We have a logger now. Someone hits your /todos endpoint. What information
 would actually be useful to log?"

 method        →  GET, POST, PUT, DELETE
path          →  /api/v1/todos
statusCode    →  200, 404, 500
duration      →  how long the request took in ms
userId        →  who made the request
correlationId →  unique ID for this specific request

❌ passwords          →  even if hashed
❌ JWT tokens         →  anyone with this token can act as that user
❌ API keys / secrets →  gives access to external services
❌ credit card numbers →  instant legal and financial liability
❌ full name + address + phone together  →  PII combination

"Imagine a user has orders. How would you design the URL to get all orders belonging to a specific user?"
GET  /users/:id/orders         →  orders belong to a user       ✅
GET  /users/:id/orders/:oid    →  one specific order of a user  ✅
